Markdown Parser Presentation

This parser is designed to convert Markdown text into HTML. It handles various Markdown elements, such as headers, paragraphs, lists, code, and others.

Property Testing

Property-based tests are used to ensure the parser's correctness. This involves generating random Markdown documents and verifying that the parser processes them correctly. The repository also includes Unit tests that check individual parser functions.

Here's an example of a property-based test, demonstrating Markdown generation and checking for consistency with the generated text: 


it "parses valid inlineElement based on generated input and matches merged structure" $
  withMaxSuccess 1000 $
    property $
      forAll genInlineElement $ \expected -> do
        testParser parseInlineElement (renderInlineForTest expected) `shouldParse` expected

Performance Metrics

To evaluate the parser's performance, the number of char function calls during parsing is measured. This gives an idea of how many "character" operations are required to process a Markdown document.

The graphs generated from the measurements show the relationship between the metric (ratio of char calls to document size) and the depth and size of the Markdown structure. Initial observations suggest that performance scales linearly with size and probably exponential with depth.

The graphs illustrate the following:

  • Ratio vs Depth (by Size): This graph shows how the ratio of character calls to document length changes as the depth of the Markdown structure increases, for different document sizes.
  • Ratio vs Size (by Depth): This graph shows how the ratio changes as the document size increases, for different depths of the Markdown structure.

These metrics help to understand how the parser's performance scales with the complexity and size of the input Markdown.

Command Line Interface (CLI)

The CLI provides a convenient way to use the parser. The following options are supported:

  • -i, --input FILENAME: Specifies the input Markdown file (default: stdin).
  • -o, --output FILENAME: Specifies the output HTML file (default: stdout).
  • -v, --verbose: Enables verbose output (parser logs).
  • -m, --metrics: Show parser performance metrics.

Example of using the CLI: 


stack run -- input input.md --output output.html --verbose --metrics
