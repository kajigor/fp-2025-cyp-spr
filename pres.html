Markdown vs HTML

Markdown: A lightweight markup language with plain text formatting syntax. Easy to read and write.

HTML: The standard markup language for creating web pages. Describes the structure and content of web pages.

Our project converts Markdown text into structured HTML.

The Parser

Implemented using the Megaparsec library.

Parses Markdown input into an Abstract Syntax Tree (AST) represented by the MarkdownDoc data type.

Key components include parsers for headers, lists, emphasis, code blocks, etc.

data MarkdownElement
  = Header Int [InlineElement]
  | Paragraph [InlineElement]
  | CodeBlock (Maybe Text) Text
  | BulletList [[InlineElement]]
  | NumberedList [[InlineElement]]
  | HorizontalRule
  | EmptyLine
  | BlockQuote [MarkdownElement]

The Parser (cont.)

Handles both block-level and inline elements.

Uses combinators from Megaparsec to build complex parsers from simpler ones.

parseMarkdownDoc :: Parser MarkdownDoc
parseMarkdownDoc = withLogs "parseMarkdownDoc" $ do
  content <- many parseMarkdownElement
  return $ MarkdownDoc content

The Utils.hs module provides common parsing utilities.

Logging

The Logger.hs module provides simple logging functionality.

Includes functions for informational (logInfo) and debug (logDebug) messages.

logInfo :: MonadIO m => Text -> m ()
logInfo msg = liftIO $ hPutStrLn stderr $ "[INFO] " ++ T.unpack msg

logDebug :: MonadIO m => Text -> m ()
logDebug msg = liftIO $ do
  enabled <- isDebugEnabled
  if enabled
    then hPutStrLn stderr $ "[DEBUG] " ++ T.unpack msg
    else return ()

Logging (cont.)

Specific logging for parser calls and results is included for debugging parsing logic.

Debugging can be enabled via the command-line interface.

logParserCall :: String -> String -> IO ()
logParserCall name input = do
  enabled <- isParserDebugEnabled
  if enabled
    then do
      hPutStrLn stderr $ "[PARSER] Call: " ++ name
      hPutStrLn stderr $ "[PARSER] Input: \"" ++ input ++ "\""
    else return ()

HTML Rendering

The HTML.hs module is responsible for converting the Markdown AST into an HTML structure.

Defines data types for HTML elements and attributes.

data HtmlElement
  = HtmlTag Text [HtmlAttribute] [HtmlElement]
  | HtmlText Text

data HtmlAttribute = HtmlAttribute Text Text

HTML Rendering (cont.)

The markdownToHtml function traverses the Markdown AST and generates the corresponding HTML structure.

The renderHtml function serializes the HTML structure into a final HTML string.

markdownToHtml :: MarkdownDoc -> HtmlDoc
renderHtml :: HtmlDoc -> Text

CLI Utility

The CLI.hs module provides a command-line interface using the optparse-applicative library.

Allows specifying input and output files (or using stdin/stdout).

data Options = Options
  { optInput    :: Maybe FilePath
  , optOutput   :: Maybe FilePath
  , optVerbose  :: Bool
  }

CLI Utility (cont.)

The run function orchestrates the entire process: reading input, parsing, converting, and writing output based on the provided options.

Includes verbose mode for logging.

run :: Options -> IO ()

Usage: stack run Md2HtmlParser-exe -- -i input.md -o output.html -v
